### 금지 목록 — 이걸 하면 학생 프로젝트 사이트가 된다

아래 패턴이 보이면 **즉시 다시 설계**하라. 변명 없음.

```
❌ 아이콘 + 제목 + 설명 카드 그리드  →  SaaS 랜딩 클리셰
❌ 좌텍스트/우폰 레이아웃 반복       →  Framer 템플릿
❌ "핵심 기능" 섹션에 6칸 카드       →  스타트업 홈페이지 복붙
❌ 다크 배경 섹션 + 흰 카드들        →  어디서나 보이는 패턴
❌ 데이터가 문장 안에 묻힘            →  숫자가 죽는다
❌ 모든 섹션이 같은 구조             →  지루함
❌ 그라디언트 배경                   →  아마추어 신호
❌ 인사이트 = 아이콘 + 카드 3개      →  내용보다 틀이 앞섬
```

---

### 시니어급이란 — 타이포그래피가 디자인이다

학생 사이트와 시니어 사이트의 차이는 **타이포그래피 하나**다.

```
학생급:  박스·카드·그리드가 구조를 만든다
시니어급: 글자의 크기 대비·여백·위계가 구조를 만든다. 박스는 없거나 최소.
```

**구체적으로:**
- `INSIGHT 1` → 섹션 하나가 통째로 한 인사이트. 헤드라인 80px, 여백 광활.
- `62.5%` → 문장 안에 묻히면 죽는다. 뷰포트 폭의 절반 크기 타이포로.
- `68명 · 6명 · 4명` → 숫자가 주인공. 레이블이 작은 주석.
- 인사이트 3개 → 카드 3장 ❌. 각각 독립 섹션, 한 화면 = 한 메시지.
- 전략 S1~S7 → 설명 나열 ❌. 큰 번호 + 한 줄 + 증거.

---

### 섹션마다 다른 레이아웃 논리

반복은 지루함이다. 각 섹션은 고유한 구성 논리를 가진다.

```
HERO        좌텍스트 / 우폰 (넓은 여백, 깊이감)
SERVICE     사진 3장 풀블리드 분할
THE TWIST   텍스트 중앙 정렬, 한 문장이 주인공
RESEARCH    숫자 3개 카운터업 → 아래 설명
INSIGHTS    각 인사이트 = 독립 섹션, 번호 대형
AS-IS       왼쪽 스크린샷(흑백) / 오른쪽 POINT 텍스트
CONCEPT     풀블리드 사진 + 텍스트 오버레이
STRATEGY    세로 아코디언형 (번호+제목+한줄), 열리면 미니렌더
DUAL SYSTEM 가로 비교 테이블 (좌iOS/우Android)
AI HARNESS  다이어그램 + 숫자 수치들
```

### 웹 네이티브 원칙

- **마진**: `clamp(80px,10vw,160px)` — 플러스엑스식 넓은 여백. 폰 잘림 없음.
- **올 라이트**: #F5F5F5 / #FFFFFF. 다크 0. (OUTRO 블루 풀블리드 1장 예외)
- **워터마크 배경 타이포**: 없음.
- **모션**: parallax + 카운터업. 눈에 띄는 등장이 아니라 **자연스러운 등장**.
- **폰 테두리**: 5px 얇게. Android 하단바 없음.

### 톤 결정 (확정)
- 배경: **올 라이트** #F5F5F5 / #FFFFFF 카드. 다크 섹션 0. (OUTRO만 브랜드 블루 풀블리드 1장 허용)
- 워터마크 배경 타이포: **사용 안 함** (비핸스처럼 깔끔).
- 모션: **중간** — parallax + 폰 회전 + 카운터업.
- 폰 목업 테두리: **얇게** (베젤 패딩 6px 내외, 과한 두께 금지).
- Android 목업: **하단 제스처 바 넣지 않음**.
- 페르소나: **시니어 5060 (Primary) / 2030 청년 (Secondary)**.

## 0. TL;DR — 우리가 내릴 결정

두 레퍼런스는 **다른 두 가지 장르**다. 섞되, 역할을 명확히 나눈다.

| | 플러스엑스 (T우주) | 비핸스 (코레일톡) |
|---|---|---|
| 장르 | 풀스크린 시네마틱 쇼케이스 | 정통 UX 케이스 스터디 |
| 컨테이너 | 100vw 풀블리드 | 1280px 중앙 정렬 |
| 강점 | 임팩트, 브랜드 몰입 | 사고 과정의 논리, 데이터 설득력 |
| 약점 | 목업 물량 폭증, "왜?"가 약함 | 임팩트가 평범, 외울 장면이 없음 |
| 심사 적합 | 비주얼 점수 | **논리·완성도 점수 (수상 핵심)** |

### 채택 전략 (하이브리드)
- **뼈대 = 비핸스**: 데스크리서치 → 유저리서치 → 인터뷰 → 방향성 → 컨셉 → 디자인시스템 → 화면별 TO-BE → 마무리. 논리적 차곡차곡.
- **연출 = 플러스엑스**: 섹션 단위 풀블리드 전환, 거대 배경 타이포, 넓은 여백, 명암 교차, 3D/목업을 띄우는 floating 레이아웃.
- **차별화 = 듀얼 디자인 시스템**: 비핸스 코레일톡엔 없는 섹션. iOS HIG vs Android MD3 나란히 비교 = 우리만의 수상 킥.
- **AI 기술력**: T우주가 "Generate AI Images / AI 활용 운영" 섹션으로 기술력을 자랑한 것처럼, 우리도 "바이브코딩 + 듀얼 분기 아키텍처 + 휴리스틱 자동검증"을 한 섹션으로 박는다. (UX포트폴리오 + AI기술능력 동시 충족)

## 1. 레퍼런스 A — 플러스엑스 / T우주 (페이지별 정밀 크리틱)

### A-1. Hero (p1)
- **구성**: 3D 렌더 아이폰 1대를 우측에 비스듬히 세움. 흰 계단형(스텝) 배경에 폰이 놓임. 좌측·중앙 텅 빈 여백. 로고만 폰 화면 중앙에.
- **잘한 점**: 압도적 여백. 텍스트 0, 오브제 1개로 "고급" 신호. 스크롤 유도 화살표를 우상단 고정.
- **크리틱**: 첫 화면에 프로젝트명/한 줄 카피가 없어 맥락 0. 우리는 **로고 + 강렬한 한 줄 카피("강릉시 지역화폐, 다시 태어나다")는 반드시** 넣는다. 폰은 1대보단 iOS·Android 2대 나란히가 우리 컨셉(듀얼)에 맞음.
- **차용**: 계단형/플로팅 배경, 우상단 스크롤 인디케이터, 극단적 여백.

### A-2. Overview (p2)
- **구성**: 좌측에 거대 3D 둥근사각 아이콘(브랜드 심볼) 떠 있음 + 그림자. 우측에 `OVERVIEW` 이서브로우 → 영문 대형 H2("We aim to find the identity...") → 국문 본문 3줄 → 작은 주석(*팔로업 기간 어쩌고).
- **잘한 점**: **영문 대형 헤드라인 + 국문 본문**의 이중 타이포가 세련됨. 우상단 주석으로 솔직함(런칭본과 차이) 표현 = 신뢰.
- **크리틱**: 3D 아이콘은 우리가 만들기 부담. 우리는 **앱 아이콘 목업 또는 SVG 심볼**로 대체. 영문 헤드라인 패턴은 그대로 차용 ("We rebuild the local currency experience" 류).
- **레이아웃 공식**: `좌 비주얼 50% / 우 텍스트 50%`, 세로 중앙 정렬, 좌우 큰 패딩.

### A-3. Market Research (p3-4)
- **구성**: 상단 좌측 정렬 헤드라인("구독 서비스는 점점 확장... 친숙도는 떨어지고") + 본문. 하단에 **연도 타임라인(1920→2015→2018→2021→2024)** 좌측, 우측에 두 칼럼(Familarity / Spectrum) 막대가 연도별로 길어짐. 마지막 2024만 브랜드 블루로 강조 + "현재의 T 우주" 말풍선.
- **잘한 점**: **회색 막대들 사이 마지막 하나만 컬러** = 시선 집중의 정석. 타임라인으로 "시장은 이미 이만큼 컸다"를 시각화.
- **크리틱**: 막대가 placeholder처럼 비어 추상적. 우리는 **실제 수치(지역화폐 발행액, 사용자수, 가맹점수 등)를 막대/도넛에 박아** 비핸스식 데이터 설득력으로 보강. (회색+1컬러 강조 규칙은 그대로 차용)

### A-4. Client Interview (p5-6)
- **구성**: `CLIENT INTERVIEW` → 헤드라인 → 본문. 아래에 거대 흐릿 배경타이포 `Goal` `Strength` `Differentiation` 깔고, 그 위에 **알약형 흰 칩**("AI를 활용한 서비스 발굴", "다양한 브랜드와의 제휴 파워", "지불 대비 좋은 혜택")이 좌우 지그재그로 떠다님.
- **잘한 점**: 인용/키워드를 **둥근 pill로 흩뿌리는 연출**이 가볍고 모던. 배경 대형 워터마크 타이포로 챕터 구분.
- **크리틱**: 강릉페이엔 "클라이언트(강릉시) 인터뷰"가 실제로 없으면 억지. 있으면 차용, 없으면 **이 pill-floating 패턴을 "프로젝트 목표/포지셔닝 키워드"** 섹션에 재활용. (배경 워터마크 타이포 = 핵심 차용 1순위)

### A-5. User Research (p8-10)
- **구성**: `USER RESEARCH` → 긴 헤드라인 → 본문. 아래 **흰 카드 6장 staggered 2칼럼**. 카드마다 우상단 연한 보라 `USER 01~06` 라벨 + 큰 인용문("T 우주가 무엇인지 모르겠어요", "구독 상품에 어떤 혜택이 있는지 모르겠어요", "상품이 광고처럼 느껴져요"...).
- **잘한 점**: **유저 보이스를 큰 따옴표 카드로** = 페인포인트가 즉각 와닿음. 좌우 카드 높이/위치 어긋나게(staggered) 배치해 리듬감.
- **크리틱**: 카드가 비어(인용문만) 데이터 빈약. 우리는 **인용문 + 페르소나 일러스트/아이콘 + 응답률(%)** 조합 (비핸스 코레일톡 인터뷰 카드 방식과 결합).
- **차용 1순위**: staggered 카드 그리드 + `USER 0X` 라벨 + 큰 인용 타이포.

### A-6. Direction (p11-12)
- **구성**: 다크 배경 전환. `DIRECTION` → 헤드라인("고객의 이해를 기반... UX 방향성 4가지"). 아래 **다크 카드 4장**(IMPRESSION / EXPLORATION / SUBSCRIPTION / BENEFIT), 각 카드에 영문 대제목 + 국문 소제목 + 설명. 카드들이 높이 어긋나게 배치.
- **잘한 점**: **라이트→다크 전환**으로 "전략 챕터" 진입을 명확히 알림. 4방향성을 카드로 라벨링.
- **크리틱**: 우리도 UX 방향성 N가지를 다크 카드로. 단 강릉페이 맥락(시니어 접근성, 듀얼 플랫폼)에 맞는 방향성으로. **다크 섹션 = 전략/원칙 챕터**라는 규칙 확립.

### A-7. UX/UI Concept (p13-14)
- **구성**: 풀스크린 **눈동자 클로즈업 사진**(좌 파란눈/우 갈색눈) + 좌하단 흰 텍스트("FROM USER INSIGHT TO T UNIVERSE"). 다음 화면 다크 배경에 초대형 `INSIGHT` 타이포 + 폰 목업이 글자 사이로 솟음.
- **잘한 점**: **풀블리드 감성 사진 + 초대형 타이포**의 시네마틱. "From A To B" 카피 공식. 글자 뒤에서 폰이 등장하는 레이어링.
- **크리틱**: 감성 사진은 강릉(바다/커피거리/경포 등 지역성)으로 치환하면 **강릉 로컬리티**까지 챙김 = 지역화폐 프로젝트에 완벽. 초대형 타이포+목업 레이어링은 핵심 장면으로 차용.

### A-8. 핵심 기능 쇼케이스 (p15-17, 24-40)
- **구성**: 거대 배경 워터마크 타이포(`IMPRESSION` `EXPLORATION` `SUBSCRIPTION` `BENEFIT`) 위에 폰 목업 1~2대 비스듬히, 주변에 3D 오브제(코인/지갑/카드)와 알약형 캡션이 떠다님. 좌측 `라벨 + 영문H2 + 국문본문` 고정.
- **반복 공식**:
  - `좌측 텍스트 블록(라벨/영문제목/본문) + 우측 폰 목업` 또는 그 반대(지그재그)
  - 섹션 배경에 흐린 대형 워터마크 타이포
  - 컬러 배경 블록(레드/블루/퍼플)으로 기능별 색 구분
  - 3개 폰을 STEP으로 나란히(p34 상품상세, p55-56 예매 플로우 차용 가능)
- **크리틱**: **이 패턴이 우리 "핵심 화면 TO-BE" 섹션의 메인 틀**. 우리는 실제 목업(스크린샷)을 넣으니 더 강력. 다만 3D 오브제는 부담 → 생략하고 목업+캡션 pill로 충분.

### A-9. Design System (p18-23)
- **구성**: `UI ELEMENTS`/폰트 비교(좌 T Universe sans / 우 Pretendard, 거대 `Aa`). 컬러칩(T Blue #3617CE, Gray 단계, B&W). 그리드 위 로고 구성(T/Heart 심볼 도형 분해). 아이콘 셋 그리드(DISCOVER/SUBSCRIPTION/... 15종).
- **잘한 점**: **거대 `Aa`로 폰트 보여주기**, 컬러를 큰 색면 칩으로, 아이콘을 균일 그리드로 = 디자인시스템 섹션의 교과서.

## 2. 레퍼런스 B — 비핸스 / 코레일톡 (페이지별 정밀 크리틱)

> 이쪽이 **수상 구조의 정석**. 컨테이너 1280px, 흰 배경, 코레일 블루(#006AFF) 포인트.

### B-1. Hero (p42)
- **구성**: 좌측 거대 국문 타이포("마음을 잇다") + 하단 카피. 우측 폰 목업 2대 비스듬+코레일 로고. 하단 파란 블록에 `PROJECT DETAIL`(TOOL/PERIOD/SERVICE) + 우측 프로젝트 한 줄 소개. 태그 칩(Booking service / Brand Experience).
- **잘한 점**: **첫 화면에 프로젝트 메타데이터(툴/기간/서비스)를 박아** 케이스스터디임을 즉시 선언. 슬로건 타이포 강렬.
- **차용 1순위**: Hero 하단 `PROJECT DETAIL` 메타 블록 = **반드시 차용** (수업명/팀/기간/툴/역할). 플러스엑스엔 없는 정보. 심사위원 첫인상.

### B-2. Service Analysis (p43)
- **구성**: `SERVICE ANALYSIS` → "여행·교통 어플 코레일톡, 사용해 보신 적 있으신가요?" → 본문. 아래 **감성 사진 카드 3장**(BOOKING/TRAVEL PACKAGE/MaaS SERVICE), 사진 위 흰 텍스트 오버레이.
- **잘한 점**: 서비스가 뭔지를 **질문형 헤드라인 + 사진 카드 3분할**로 부드럽게 도입.
- **차용**: "강릉페이, 써보신 적 있으신가요?" 도입. 강릉 지역 사진 3장(가맹점/시장/관광)으로 서비스 맥락.

### B-3. Current/AS-IS (p44)
- **구성**: `CURRENT` → "그러나... 사용하기 너무 어려워요"(어려워요 컬러 강조). 좌측 본문. 중앙 실제 앱 스크린샷 4장 겹침. 우측 `POINT 1/2/3`(컨텐츠 정리 부족 / 아이덴티티 결여 / 위계 없는 정보) 각 설명.
- **잘한 점**: **POINT 1·2·3 넘버링 문제 진단**. 실제 스크린샷으로 증거 제시. 본문 키워드만 컬러 강조.
- **차용 1순위**: AS-IS 문제를 **번호 매긴 POINT**로. 실제 강릉페이 스크린샷 + 빨강/강조 표시(휴리스틱 위반 6건과 연결). 헤드라인 부분 컬러 강조 기법.

### B-4. Desk Research (p45)
- **구성**: `DESK RESEARCH` → "늘어나는 모빌리티 서비스 수요와 코레일톡의 방향"(키워드 컬러). 본문. 아래 **데이터 카드 3장**: ① 바차트(일일 이용객 24만6천), ② 도넛(온라인예매 88.2%), ③ 버블차트(다운로드 1위). 하단 파란 `KEY INSIGHT` 박스.
- **잘한 점**: **실수치 + 차트 3종 + KEY INSIGHT 요약 박스**. 데스크리서치의 완성형. 출처 주석까지.
- **차용 1순위**: 우리도 지역화폐 시장 데이터(발행규모/가맹점/캐시백 등)를 **바·도넛·버블 3카드 + KEY INSIGHT 파란박스**. 출처 주석(통계청/행안부/강릉시) 필수. = 비핸스 수상 핵심 섹션.

### B-5. User Research (p46)
- **구성**: `USER RESEARCH` → "코레일톡, 언제 어떻게 사용하시나요?" → 본문 + 우측 조사개요(기간/대상 N명). 아래 **3카드**: ① 가로 바(사용 상황 67/40/25..), ② 도넛(예매외 서비스 모름 88.3%), ③ 페인포인트 6칩.
- **잘한 점**: 우상단 **조사 개요 메타(기간·표본수)** = 신뢰. Q. 형식 질문 헤드라인. 정량(바/도넛)+정성(칩) 혼합.
- **차용**: 설문 결과를 동일 포맷. 우상단에 "2026.0X.XX / 강릉페이 사용자 N명" 메타.

### B-6. Interview (p47)
- **구성**: `INTERVIEW` → "실제 사용자들은 어떤 불편함을..."(컬러 강조). 조사 메타. 아래 **유저 인터뷰 카드 4장**: 일러스트 + 큰 인용("혼자서도 쉽게 예매하고 싶어요") + 작은 보충 + `USER A (60세)` 라벨. 2x2 staggered.
- **잘한 점**: **연령 라벨 + 일러스트 + 인용**으로 페르소나 생생. (플러스엑스 p8 카드의 비핸스 버전 — 더 정보 풍부)
- **차용 1순위**: 우리 인터뷰 카드 = 일러스트(or 아이콘) + 인용 + 연령/유형 라벨. **시니어 사용자** 강조하면 강릉페이 듀얼시스템 명분과 연결.

### B-7. Direction (p48)
- **구성**: `DIRECTION` → "주요 포인트 4가지를 통해 편안한 여행을". 본문. 아래 **흰 카드 4장**: 각 상단 컬러 태그(플로우 개선/몰입과 집중/탐색 여정 강화/여행의 시작과 끝) + 제목 2줄 + 일러스트 + 하단 설명.
- **잘한 점**: 방향성을 **태그+제목+일러스트+설명** 4단 구조 카드로 균일하게. (플러스엑스 p11 다크카드의 라이트 버전)
- **차용**: UX 방향성 4가지를 이 카드 포맷. 우리는 다크로 갈지 라이트로 갈지 = 위 질문 1번 답에 따라.

### B-8. UX Writing / Brand Voice (p49)
- **구성**: `UX WRITING` → "여행을 함께하는 동반자처럼, 쉽고 간단한 멘트로". 좌측 BRAND VOICE 슬라이더 카드(공손한↔투박한 등 5축 토널 슬라이더). 우측 CHANGE(기존vs개선 멘트 비교 칩들).
- **잘한 점**: **보이스&톤을 슬라이더로 시각화** + Before/After 멘트 비교 = 디테일 끝판왕. 심사 가산점 요소.
- **차용**: 강릉페이도 UX라이팅 개선(예: 환불 무피드백→친절 스낵바 멘트) 있으면 이 Before/After 칩 포맷. 토널 슬라이더는 여력되면.

### B-9. Concept (p50)
- **구성**: `CONCEPT` → "여행가는 날처럼 화창한 날씨를 담아... 여정의 설렘". 본문. 아래 **감성 사진 3장**(하늘 WIDE·CLEAR / 바다 ROUND·COMFORT / 우주 DEEP·IMMERSIVE) + 사진 위 텍스트.
- **잘한 점**: 브랜드 컨셉을 **자연 이미지 3장 + 키워드**로 정서적 정당화. 컬러 선택 근거(코레일 블루=하늘/바다)를 사진으로.
- **차용**: 강릉페이 블루(#1D4ED8)의 근거를 강릉 바다/하늘 사진으로. "신뢰·청량·지역" 키워드 3장.

### B-10. Design System (p51)
- **구성**: `DESIGN SYSTEM` → "편안하고 시원한 코레일 블루를 사용했어요". **좌측 Pretendard 타이포 스케일 표**(H1 24 SemiBold...), **우측 컬러 팔레트**(KORAIL BLUE #006AFF / Dark / Light 4단 / Gray 단계 / B&W).
- **잘한 점**: **타이포 스케일 표 + 컬러 시스템**을 깔끔한 카드로. 실무 디자인시스템 그대로.
- **차용 1순위**: 우리 `DESIGN.md`의 토큰(타이포 스케일/컬러/라운드)을 이 포맷으로 시각화. **여기에 우리만의 듀얼(iOS/Android) 비교를 덧붙이면 차원이 다른 디자인시스템 섹션** 완성.

### B-11. 화면별 TO-BE (p52-58)
- **구성**: `SERVICE`/`MAIN`/`TICKET BOOKING`/`MY PAGE` 등 기능별 섹션. 좌측 라벨+헤드라인(키워드 컬러)+본문 + 하위 라벨(POP-UP/LOCATION CHECK 등) 설명. 우측/중앙 폰 목업. **STEP 1~8 예매 플로우를 폰 나란히로** 펼침. AS-IS 작은 흑백 + TO-BE 컬러 비교.

- **차용 1순위**: 우리 핵심 4기능(카드신청/충전/환불/QR결제)을 이 포맷. **AS-IS 흑백 + TO-BE 컬러 비교**는 강력 → 휴리스틱 개선 6건과 직결. STEP 펼침은 플로우 있는 기능(예: 충전→결제)에.

### B-12. Outro (p59)
- **구성**: 파란 블록 풀블리드 `THANK YOU FOR WATCHING` 대형 타이포 + 폰 목업 2대 + 슬로건 + 로고 + 카피라이트.

## 4. 사이트 정보구조 (IA) — 확정 섹션 순서

> 전시회용 풀버전(길다). 기능 설명 파트 + UX 전략 파트 둘 다 포함.

```
[UX 코어 — 논리 흐름, 끊김 없이]
00. NAV / 01. HERO / 02. OVERVIEW / 03. SERVICE
04. THE TWIST     ★리서치 도중 앱 리뉴얼 → 끝까지 살아남은 진짜 문제만 선별

[기능·기술 — 전부 뒤에 크게 (우리의 핵심 무기)]

17.  DUAL SYSTEM   iOS HIG × Android MD3 컴포넌트 비교 (업계 최초 킥)
18.  AI HARNESS    병렬 에이전트·8문서·자기검증 (디지털인문예술 기술×인문)
19.  PROCESS       Phase 1~3 + 휴리스틱 위반 6건 개선 (간략)

[마무리]
20. USER TEST (예정 자리) / 21. PROTOTYPE / 22. CREDITS+OUTRO (블루 풀블리드, 마카모예 6인)
```

### 구조 철학 (C안 확정)
- UX 코어(리서치→전략)는 한 호흡으로. STRATEGY에서 미니 프리뷰로 호기심만.
- 기능·기술은 뒤 독립 챕터에서 크게크게. 전략↔기능 인과 연결 유지.
- **발표 축약 = 16B·17·18·19 off → UX 코어만 남음.** 전시회 = 풀버전.

### 목업 출처 규칙 (확정)
- **AS-IS**: 사용자가 직접 첨부한 현행 강릉페이 스크린샷 사용.
- **TO-BE**: 실제 프로토타입 파일(client/)에서 따온 화면. (미니렌더 컴포넌트 재활용)
- 그 외 장식용 폰 프레임: CSS로 생성 (테두리 얇게, Android 하단바 없음).
- 사용자에게 별도 요청할 목업: SETUP.md의 "사용자 준비물" 목록 참조.
