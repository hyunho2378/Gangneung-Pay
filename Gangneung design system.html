03 — Patterns

패턴

컴포넌트 조합으로 만들어지는 반복 사용 패턴입니다.

Form Pattern
입력 필드 + 금액 칩 + 하단 고정 CTA 버튼 조합. 충전 화면에서 사용.
1
QuickAmountChip으로 금액 프리셋 선택
2
직접 금액 입력 (iOS outline / Android filled)
3
Button variant="filled" size="lg" (하단 고정)
List Pattern
MY 페이지, 설정 화면의 메뉴/토글 행 구성 패턴.
1
SectionHeader (회색 배경, bold)
2
MenuRow: 라벨 + ChevronRight + 값 (minHeight 44px)
3
ToggleRow: 라벨 + ToggleSwitch (minHeight 44px)
Auth Pattern
결제/환불/카드뒷면 확인 시 생체인증 오버레이 플로우.
1
바텀시트 확인 → "신청하기" 탭
2
PaymentAuthOverlay 노출 (iOS 중앙 / Android 하단)
3
RAF 루프 로티 재생 → onComplete() → 결과 반영
Coach Mark Pattern
첫 진입 시 기능 진입점을 단계별로 안내하는 온보딩 패턴.
1
OnboardingContext로 seen 여부 체크 (session-scoped)
2
스포트라이트 + 말풍선 순차 노출 (cardApply → charge → refund)
3
다음/건너뛰기 → markSeen() → 다음 단계
Empty State Pattern
데이터 없음 상태에서 사용자 행동을 유도하는 패턴.
1
SVG 일러스트 (빈 문서 + X 아이콘)
2
안내 메시지 (gray[700])
3
선택적 CTA: Button variant="filled" size="sm" fullWidth={false}
Feedback Pattern
액션 완료 후 사용자에게 결과를 전달하는 피드백 패턴.
1
iOS: 전용 완료화면 (체크 SVG + 잔액 요약)
2
Android: Snackbar "N원 환불이 완료됐어요" (2.8s)
3
showSnackbar()는 AppContext 전역, iOS에서 호출해도 null 반환
