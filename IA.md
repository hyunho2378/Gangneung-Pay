## 핵심 변경 사항

### 바텀 네비게이션 (5개)
```
[홈] [결제매장] [QR결제 중앙] [이용내역] [MY]
```
- 햄버거 메뉴 완전 삭제 → MY 탭으로 흡수
- 생활편의 탭 삭제 (숨김)
- 소통참여 탭 삭제 (숨김)
- QR결제를 중앙 강조 위치로
- 이용내역을 기존 QR결제 자리로

### 상단 헤더
```
[로고 강릉페이] [큰글씨] [검색] [알림종]
```
- 햄버거 자리에 알림 종 모양 SVG
- 검색 누르면 매장이 아닌 검색 전용 페이지로
- 검색 페이지에 자주 찾는 키워드 노출

## 신규 IA 트리

```
앱 진입
├── 신규 사용자 (카드 없음)
│   ├── 홈 진입 → 카드 신청 섹션 스포트라이트
│   ├── 카드 신청 화면 (첨부 이미지 2, 3 디자인)
│   ├── 신청 완료 → 몇 초 후 배송 완료 알림
│   ├── 카드 등록 화면
│   └── 등록 완료 → 충전 코치마크
│
└── 기존 사용자
    └── 메인 홈 진입

바텀 네비게이션
├── 홈 (/)
│   ├── 잔액 카드 (카드 SVG 클릭 시 뒷면 CVC, FaceID 또는 지문 인증)
│   ├── 카드 신청 섹션 (신규 사용자만)
│   ├── 캐시백 안내 섹션 (카카오페이 결제 안내 통합)
│   └── 카카오페이 가이드 진입 (캐시백 섹션 내)
│
├── 결제매장 (/store)
│   ├── 구글 지도 (변경)
│   ├── 자주 가는 곳 (신규)
│   └── 매장 상세
│
├── QR결제 (/qr) — 중앙 강조
│   └── 실제 카메라 스캔
│
├── 이용내역 (/history) — 기존 QR 자리
│   ├── 거래 기록 (환불 사례 포함)
│   ├── 환불 버튼 → 환불 가능 충전내역 즉시 표시
│   └── 카드 관리 → MY로 이동
│
└── MY (/my) — 신규
    ├── 내 카드 보기
    ├── 카드 관리
    ├── 교통카드 기능 추가
    ├── 회원정보 변경
    ├── 비밀번호 변경
    ├── 본인확인 정보
    ├── 분실신고/재발급
    ├── 고객센터
    ├── 설정
    ├── 이용 약관
    └── 가맹점 신청

상단 헤더
├── 검색 (/search) — 자주 찾는 키워드
└── 알림 (/notification) — 종 아이콘
```

## 페이지별 변경 사항

### HomePage
유지:
- 카드 SVG 디자인, 브랜드 색상

수정:
- 진행단계 스텝퍼 삭제 (코치마크로 대체)
- 카드 신청 섹션 위로 이동
- 캐시백 안내 별도 섹션화
- 카카오페이 결제 안내 → 캐시백 섹션에 통합
- "강릉 전역에서 사용하세요" 민트 → primary 색상
- 사랑통기부통 삭제 (숨김)
- 지원금 랭킹 삭제 (숨김)
- "충전하기, 이용내역, 강릉여행 이용안내" 편집 섹션 삭제
- 맨 아래 카카오페이 배너 삭제
- 카드 SVG 클릭 시 뒷면 모달 (CVC, 카드정보)
  iOS: Face ID, Android: 지문 인증 후 노출

신규:
- 신규 사용자 카드 신청 플로우
- 카드 등록 전 "카드 없어요" 상태
- 카드 등록 후 실제 카드 + 충전 시 금액 반영

### StorePage
- 카카오 지도 → 구글 지도
- 자주 가는 곳 섹션 추가

### QRPage
- 바텀 네비 중앙 위치
- 실제 카메라 스캔 (html5-qrcode)

### HistoryPage
- 기존 QR 자리로 이동
- 환불 사례 포함
- 환불 버튼 → 환불 가능 충전내역 즉시 표시
- 카드 관리 섹션 → MY로 이동

### MyPage (신규)
햄버거 메뉴 항목 흡수.
제외:
- 알림 (헤더 종 아이콘으로 분리)
- 지인에게 소개하기

### 신규 페이지
- SearchPage (/search)
- CardApplyPage (/card-apply) — 이미지 2, 3 기준
- CardBackModal — 카드 뒷면 정보

## 코치마크 / 온보딩 흐름

신규 사용자:
```
스플래시 → 홈 진입
↓
1. 카드 신청 섹션 스포트라이트
   "강릉페이 카드를 신청하면 시작할 수 있어요"
↓
2. 신청하기 → CardApplyPage (이미지 2, 3 디자인)
↓
3. 신청 완료 → 몇 초 후 배송 완료 알림
↓
4. 카드 등록 화면
↓
5. 등록 완료 → 충전 코치마크
   "이제 충전해서 사용해보세요"
↓
완료
```

기존 사용자: 코치마크 표시 안 함
건너뛰기 한 번 누르면 영구 표시 안 함
(localStorage 금지이므로 React Context로 세션 내 유지)

## 버그 수정 항목

1. 코치마크가 늘어남 (앱 고정형 깨짐)
   → max-width 430px 강제, position absolute

2. 건너뛰기 버튼 스크롤 시 위치 이상
   → position fixed, viewport 기준

3. iOS Safari에서 충전 버튼 아래 잘림
   → safe-area-inset-bottom 적용
   → 100vh 대신 100dvh

4. 코치마크 한 번 건너뛰면 다시 안 뜨게
   → React Context로 세션 내 유지
