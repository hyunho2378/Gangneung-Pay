## 슬라이드 2 — HIG 준수 설계 결정: QR 탭 재배치

### 탭바에서 QR결제를 뺀 이유

**Apple Human Interface Guidelines 원문:**
> "Avoid using a tab bar for actions. Use a toolbar to provide buttons that act on elements in the current view."

| 항목 | 일반 핀테크 앱 | 강릉페이 (HIG 준수) |
|------|-------------|----------------|
| QR결제 위치 | 탭바 중앙 (액션을 탭에 배치) | **잔액 카드 액션 버튼** (컨텍스트와 일치) |
| 탭바 책임 | 네비게이션 + 액션 혼재 | **네비게이션 전용** |
| 액션 영역 | 없음 | **잔액 카드 3버튼** (충전 / 환불 / QR결제) |
| 빈 탭 슬롯 | — | **지원금·혜택** (강릉 특화 서비스 1탭 접근) |

**설계 의도:** 탭바 = 어디로 갈지 / 잔액 카드 = 무엇을 할지. 두 책임을 분리하면 사용자가 목적에 따라 직관적으로 UI 영역을 구분할 수 있음.

## 슬라이드 5 — S2 전략: 환불 동등 위계 (3중 오류 방지)

**문제:** 기존 강릉페이에서 환불 기능을 4명 전원 발견하지 못함 (관찰미션 결과)

**해결:** 홈 화면 잔액 카드에 충전·QR과 동등한 위계로 환불 버튼 배치

### 환불 신청 시 자동 3중 검증:

```
[검증 1] 잔액 >= 충전금액?
    No → 버튼 비활성 + "잔액이 없습니다"

[검증 2] 충전금액 대비 사용 비율
    1만원 이하: 80% 이상 사용했는가?
    1만원 초과: 60% 이상 사용했는가?
    No → 버튼 비활성 + "N% 이상 사용 후 환불 가능"

[검증 3] 현재 월 충전건인가?
    No → 버튼 비활성 + "환불 불가 (과거 거래)"

모두 통과 → 환불 버튼 활성
  → 바텀시트 (금액 확인 + 취소 불가 경고)
  → FaceID 인증
  → 환불 처리
```

## 슬라이드 6 — 금융 보안 인증 플로우 (FaceID)

**충전·환불 공통 인증 시퀀스:**

```
사용자 액션 (충전하기 / 환불 신청하기)
    ↓
PaymentAuthOverlay 등장 (position:fixed, zIndex:1000)
    ↓
강릉페이 간편 비밀번호 키패드 화면
    ↓ 100ms 딜레이
FaceID 모달 등장 (150×148px 검은 박스)
    ↓
RAF 244프레임 재생 — iOS easing curve (2,500ms)
    ↓ ease-out 100ms 페이드
충전/환불 완료
```

**기술 구현 포인트:**
- `dotLottieRefCallback` + `instance.addEventListener('load')` — CardBackModal과 동일 패턴
- `requestAnimationFrame` 루프: iOS easing (ease-out 30% → linear 40% → ease-in 30%)
- 폴백 타이머 2,700ms (로티 로드 실패 대비)
- 위치: `left:50% + transform:translateX(-50%) + maxWidth:390px` — 앱 고정 너비 준수

## 슬라이드 7 — 시니어 접근성 + 풀스택 시연

### 시니어 친화 접근성

| 항목 | 구현 범위 |
|------|---------|
| 큰글씨 모드 (`useTypography` 훅) | **35개 파일** 적용 |
| 최소 터치 영역 44px+ | **23개 적용 지점** 명시 |
| 아이콘 + 텍스트 병기 | BottomNavBar 전체, 결제매장 업종 필터 |
| 큰글씨 즉시 토글 | MY → 설정 → 큰글씨 모드 1탭 |

### 풀스택 시연 동선 (발표 중 라이브 시연)

```
1. 앱 실행 → 홈 화면 잔액 카드 확인
2. 충전하기 → 금액 입력 → FaceID 인증
   → Network 탭: POST /api/log 요청 확인
   → Neon 콘솔: actions 테이블에 신규 row 즉시 확인
3. 환불 → 조건 검증 → FaceID → 완료
   → 잔액 카드 실시간 갱신 확인
4. KST 변환 쿼리로 세션별 행동 통계 출력
```
